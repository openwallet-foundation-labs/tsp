# TSP Test Vectors Overview

This document summarizes the structure of the TSP test-vector set.

Detailed definitions are provided in [test-vector-instances.md](./test-vector-instances.md), [test-vector-fixtures.md](./test-vector-fixtures.md), [test-vector-fixture-inventory.md](./test-vector-fixture-inventory.md), [test-vector-complete-cases.md](./test-vector-complete-cases.md), [test-vector-case-manifests.md](./test-vector-case-manifests.md), [test-vector-spec-mapping.md](./test-vector-spec-mapping.md), and [test-vector-blockers.md](./test-vector-blockers.md).

The set is organized as a shared abstract vector and fixture catalog, plus mechanism-specific complete cases that declare applicability and freeze case-local artifacts.

## Vector Classes

### Byte-Exact

`byte-exact` vectors use one authoritative wire artifact and require exact agreement on decoded control fields or forwarding material.

Included entries:

- `BV-001` Direct RFI Decode
- `BV-002` Direct RFA Decode
- `BV-003` Direct RFD Decode
- `BV-004` Nested RFI Decode
- `BV-005` Nested RFA Decode
- `BV-006` Routed Hop Unwrap
- `BV-007` Routed RFI Final Decode
- `BV-008` Routed RFA Final Decode
- `AV-002` Ciphertext Encoding Family Decode

### Semantic-Only

`semantic-only` vectors use one authoritative wire artifact and require agreement on opened semantics, state predicates, or rejection behavior without requiring identical regenerated bytes.

Included entries:

- `SV-001` Direct Confidential Generic Message Open
- `AV-001` Confidential Control Sender-Field Presence
- `AV-003` Non-Confidential-Field Binding Review
- `SV-002` Nested Confidential Generic Message Open
- `SV-003` Routed Confidential End-to-End Open
- `SV-004` Generic Message Without Prior Relationship
- `SV-005` RFA Digest Mismatch
- `SV-006` Nested Message Without Coupled Outer Relationship

### Fixture-Only

`fixture-only` entries provide stable identity and conversation references for vectors. They do not define identity lifecycle policy, wallet layout, or local implementation objects.

Supporting fixture definitions are recorded in [test-vector-fixtures.md](./test-vector-fixtures.md) and inventoried in [test-vector-fixture-inventory.md](./test-vector-fixture-inventory.md).

## Complete Cases

This set defines three mechanism-specific complete cases:

- `HPKE-Auth`
- `HPKE-Base`
- `Sealed Box`

Each complete case realizes shared protocol assertions under one confidentiality mechanism. Complete cases do not imply cross-case byte identity or artifact identity.

`CC-001`, `CC-002`, and `CC-003` declare applicability to the shared vector and fixture catalog through case-specific manifests.

The explicit realized slices across all three complete cases are the confidential-control slices, the generic confidential-message slice, and the negative semantic-boundary slice:

- `BV-001`
- `BV-002`
- `BV-003`
- `SV-005`
- `AV-001`
- `AV-002`
- `AV-003`
- `BV-004`
- `BV-005`
- `BV-006`
- `BV-007`
- `BV-008`
- `SV-001`
- `SV-002`
- `SV-003`
- `SV-004`
- `SV-006`

Case structure is defined in [test-vector-complete-cases.md](./test-vector-complete-cases.md). Case manifests are defined in [test-vector-case-manifests.md](./test-vector-case-manifests.md).
